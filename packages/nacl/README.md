# happy-nacl — TweetNaCl NAPI as a HAR

Pure-ArkTS HAR module wrapping the NaCl subset that `slopus/happy` uses
for E2E auth + content encryption. Built and consumed as a regular
module dependency: `"happy-nacl": "file:../packages/nacl"` in the
consumer's `oh-package.json5`.

## What's inside

- `src/main/cpp/tweetnacl/` — TweetNaCl 20140427, public domain. Local
  patch adds `crypto_sign_keypair_from_seed` (note in source).
- `src/main/cpp/napi/nacl_napi.cpp` — NAPI bindings. The libsodium "easy"
  API on top of NaCl raw, plus `crypto_box_seed_keypair` matching
  libsodium's `sk = SHA-512(seed)[0..32]` derivation (NOT
  tweetnacl-js's `sk = seed`; they produce different keypairs).
- `src/main/cpp/CMakeLists.txt` — DevEco/hvigor builds this into
  `libnacl.so` automatically via `build-profile.json5`'s
  `externalNativeOptions`.
- `src/main/cpp/types/libnacl/Index.d.ts` — type stubs that flow into
  the consumer's compile graph.
- `src/main/ets/Index.ets` — ArkTS facade re-exporting the .so surface
  + symbol-size constants.

## Surface

```typescript
// either of these works in a consumer module:
import nacl from 'libnacl.so';                // direct, type-flow via transitive dep
import { secretboxEasy } from 'happy-nacl';   // named via the HAR's main entry
```

| Function | What |
|---|---|
| `secretboxEasy / OpenEasy` | XSalsa20-Poly1305 (32B key, 24B nonce, 16B tag) |
| `cryptoBoxKeypair` | random X25519 keypair |
| `cryptoBoxKeypairFromSeed` | libsodium-style: sk = SHA-512(seed)[0..32], pk = scalarmult_base(sk) |
| `cryptoBoxEasy / OpenEasy` | X25519 + XSalsa20-Poly1305 (16B MAC prepended) |
| `cryptoSignKeypairFromSeed` | Ed25519 from 32B seed |
| `cryptoSignDetached / VerifyDetached` | Ed25519 detached signatures (64B) |

`secretboxOpenEasy` and `cryptoBoxOpenEasy` return `null` on
authentication failure — the type already encodes that contract.

## Standalone smoke test (CLI, no DevEco)

`./smoke-test.sh` sanity-checks that the OpenHarmony NDK toolchain
produces a loadable `libnacl.z.so` for `aarch64-linux-ohos` from this
source tree, independent of hvigor. Defaults to the NDK shipped with
DevEco Studio. Override with `OHOS_SDK=… ./smoke-test.sh`.

## Correctness

`tests/vectors/{secretbox,box,sign}.json` are the reference fixtures
generated by `tests/vectors/generate.mjs` against `tweetnacl@1.0.3`,
the same library `happy-server` uses. The `box.json` keypair derivation
is libsodium-style on purpose — see the comment block at the top of
`generate.mjs`.

`tests/vectors/runner.ets` is dropped into a HAP/Stage app to replay
the fixtures on device against this NAPI. PoC-A2 passed 21/21
secretbox + 20/20 box + 16/16 sign rounds; PoC-A3 added the
libsodium-style box keypair fix and the fixtures were regenerated to
match.
